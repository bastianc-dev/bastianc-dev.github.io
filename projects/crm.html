01 · Contexto

Descripción

El proyecto es un CRM backend para administrar usuarios y clientes potenciales. El sistema diferencia entre administradores y empleados, y aplica permisos distintos según el rol y la propiedad de cada lead.

El objetivo fue construir una API completa y evaluable: base de datos, autenticación, reglas de negocio, pruebas, documentación y despliegue.

02 · Producto

Funcionalidades

  • Registro, inicio de sesión y refresh token.
  • Autenticación JWT y hash de contraseñas.
  • Roles de administrador y empleado.
  • CRUD de usuarios y leads.
  • Permisos por rol y por propiedad del lead.
  • Búsqueda, filtros, ordenamiento y paginación.
  • Validación de datos con Pydantic.
  • Migraciones con Alembic.
  • Manejo centralizado de errores y logging.
  • 31 pruebas automatizadas con Pytest.
  • Documentación Swagger y despliegue en Render.

03 · Ingeniería

Arquitectura

04 · Criterio técnico

Decisiones técnicas

PostgreSQL

Los datos tienen relaciones claras entre usuarios y leads, por lo que una base relacional encaja bien con el problema.

FastAPI

Facilita la validación, la documentación OpenAPI y la creación de endpoints con Python.

SQLAlchemy y Alembic

Permiten modelar el acceso a datos y versionar los cambios de la base de datos mediante migraciones.

Separación por capas

Evita concentrar toda la lógica en los endpoints y hace más fácil probar y mantener el proyecto.

05 · Resultado

Resultado

El CRM está publicado y puede revisarse desde Swagger. El repositorio incluye instrucciones de instalación, variables de entorno, arquitectura, endpoints, pruebas y decisiones técnicas.

El proyecto reúne las partes principales de una aplicación backend: lógica de negocio, permisos, persistencia, validación, testing, documentación y despliegue.
